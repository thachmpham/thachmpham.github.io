GDB: Breakpoints

1. Overview

GDB supports two types of breakpoints: software and hardware breakpoints.

Software Breakpoint.

Mechanism: To setup a software breakpoint, GDB replaces the program instruction with a trap, illegal divide or some other instructions that will cause an exception. When the exception raised, GDB catches it and stops the program.

Maximum: The number of software breakpoints is unlimited.




Permission: To set a software breakpoint, the target address must reside in a writable memory region.




Usage: Typically used for regular debugging, where performance and memory permissions are not critical.

Hardware Breakpoint.

Mechanism: To setup a hardware breakpoint, GDB stores the breakpoint address to a debug register. When the program counter $pc matches the value in the debug register, GDB stops the program.

Maximum: The number of debug registers are limited and varies depending on the architecture. For example, x86 has 4 debug registers, DR0-DR3. So, the number of hardward breakpoints is limited.

Permission: A hardware breakpoint can be placed at an address in read-only memory region.

Performance: The impact of a hardware breakpoint on program performance is smaller than that of a software breakpoint.

Usage: Typically used for read-only code or in live system, where performance and memory safety are critical.


2. Software Breakpoints

Breakpoint: Stop program whenever a point in the program is reached.

  break [-qualified] locspec
  • locspec: line number, function name, address.
  • qualified: full match.
int sum(int a, int b)
{
  return a + b;
}

int main()
{
  int a = 1, b = 2;
  int s = sum(a, b);
}
(gdb) break 3
(gdb) break sum
(gdb) break demo.c:3
(gdb) break demo.c:sum

(gdb) break -qualified exit

(gdb) disassemble sum

Regex Breakpoints: Break by regular expression.

Set breakpoints on all functions matching the regular expression regex

  rbeak regex
  rbreak file:regex
  • regex: use standard regular expression, like grep.
rbreak sum.*
rbreak demo.c:sum.*

# all functions in current program
rbreak .
# all functions in file
rbreak demo.c:.


Conditional Breakpoints: Only stop on certain conditions.

Set a conditional breakpoint.

  break locspec [-force-condition] if cond
  • cond: only stop at breakpoint when cond is true.
  • force-condition: force to enable to cond.

Change condition.

  condition breakpoint [cond]


Inferior-Specific Breakpoints: Only stop on certain processes.

  break locspec inferior inferior-id [if cond]


Thread-Specific Breakpoints: Only stop on certain threads.

  break locspec thread thread-id [if cond]


Temporary Breakpoints: Automatically deleted after the hit.

  tbreak args


Commands: Automate debugging actions when a breakpoint is hit.

Set commands.

  commands [breakpoints...]
      command
      ...
  end

Change commands.

  commands breakpoints...
      command
      ...
  end


3. Hardware Breakpoints

Set a hardware assisted breakpoint.

  hbreak [LOCATION] [thread THREADNUM]
         [-force-condition] [if CONDITION]

Set a temporary hardware assisted breakpoint.

  thbreak [LOCATION] [thread THREADNUM]
          [-force-condition] [if CONDITION]


Sample: Set hbreak at an instruction in a read-only mmap region.

File math.c

int sum(int x, int y)
{
    return x + y;
}

int sub(int x, int y)
{
    return x - y;
}

Build.

$ gcc -c math.c -o math.o

Find the offset of the .text section. The output shows it located at 0x000040.

$ readelf --section-headers --wide math.o
  [Nr] Name              Type            Address          Off    Size   ES Flg Lk Inf Al
  [ 1] .text             PROGBITS        0000000000000000 000040 00002e 00  AX  0   0  1

Find the offsets of the sum and sub functions. The output shows sum at 0x0000 and sub at 0x0020, these are relative offsets to the start of the .text section.

$ readelf --symbols math.o
Symbol table '.symtab' contains 5 entries:
   Num:    Value          Size Type    Bind   Vis      Ndx Name     
     3: 0000000000000000    24 FUNC    GLOBAL DEFAULT    1 sum
     4: 0000000000000018    22 FUNC    GLOBAL DEFAULT    1 sub

Illustration of the math.o structure.

+---------------------+
| ELF Header          | At: 0x0000
+---------------------+
| Program Header      |
+---------------------+
| .text section       | At: 0x0040
|  + sum()            | At: 0x0040 + 0x0000

|       ........      | Other sections
+---------------------+
| Section Header      |

File main.c

#include <sys/mman.h>
#include <stddef.h>
#include <fcntl.h>
#include <stdio.h>

int main()
{
    // load math.o to a new memory region
    int fd = open("math.o", O_RDONLY);
    void *addr = mmap(
        NULL,  // staring address of memory region, NULL: kernel will choose
        4096,   // length of the region
        PROT_READ|PROT_EXEC,    // protection mode
        MAP_PRIVATE,            // visible mode
        fd,                     // file to load
        0                       // offset in file
    );

    // find address of .text segment
    void *text = addr + 0x0040;

    // find address of sum and sub functions
    void *sum = text + 0x0000;
    void *sub = text + 0x0018;

    // call sum(5, 4)
    int a = ((int (*)(int, int))sum)(5, 4);

    // call sub(5, 4)
    int b = ((int (*)(int, int))sub)(5, 4);

    printf("a=%d, b=%d\n", a, b);
    // output: a=9, b=1

    return 0;
}

Build.

$ gcc -g main.c -o main

4. Management

Show breakpoints

  info break

Enable, disable breakpoints

  enable breakpoints [breakpoints]
  disable breakpoints [breakpoints]

Save, restore breakpoints.

  save breakpoints file
  source file

Delete breakpoints.

Delete all breakpoints.

  delete breakpoints

Delete all breakpoints at to locspec.
